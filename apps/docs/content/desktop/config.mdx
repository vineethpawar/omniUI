The omniUI desktop app is framework-agnostic. What makes it work for any repo is the config file at the project's root.

It looks for one of these, in order:

- `omniui.config.json`
- `omniui.config.js`
- `omniui.config.ts`
- `.omniui.json`

JSON is parsed directly. `.js` / `.ts` are parsed as a single default-exported (or `module.exports =`) object literal -- the app never executes your code.

## Minimum

```json
{
  "name": "My App",
  "writePaths": ["src/components", "src/screens"],
  "screens": [
    {
      "path": "/",
      "name": "Home",
      "sourceFile": "src/screens/Home.tsx"
    }
  ]
}
```

## Full example

```json
{
  "name": "My App",
  "readPaths": ["src", "design"],
  "writePaths": ["src/components", "src/screens", "design/tokens"],
  "figmaDumpsDir": "design/figma-dumps",
  "screens": [
    {
      "path": "/",
      "name": "Home",
      "group": "App",
      "sourceFile": "src/screens/Home.tsx",
      "figma": {
        "light": "https://www.figma.com/design/abc123/.../?node-id=10:42",
        "dark":  "https://www.figma.com/design/abc123/.../?node-id=10:43"
      },
      "description": "Hero, three feature cards, recent activity table."
    },
    {
      "path": "/settings",
      "name": "Settings",
      "group": "App",
      "sourceFile": "src/screens/Settings.tsx"
    },
    {
      "path": "/signin",
      "name": "Sign in",
      "group": "Auth",
      "sourceFile": "src/screens/auth/SignIn.tsx"
    }
  ],
  "systemPromptAppend": "All text uses Inter. Brand accent is #6D5BFF. Use the existing Button + Box primitives in src/components; don't add new dependencies."
}
```

## Fields

| Field | Required | What |
|---|---|---|
| `name` | yes | Display name for the project. Appears in the chrome and in the agent's system prompt. |
| `writePaths` | yes | Repo-relative paths the agent may write to. Everything outside is hard-denied. Required to be non-empty. |
| `screens` | yes | Per-route manifest. Each entry maps a route or page to a source file. |
| `readPaths` | no | Repo-relative paths the agent may read. If omitted, reads anywhere inside the project root. |
| `figmaDumpsDir` | no | Repo-relative folder of pre-downloaded Figma YAML/JSON. Default `design-system/figma-dumps`. |
| `systemPromptAppend` | no | Text appended to the agent's system prompt. Use for brand or convention notes. |

## Screen entry

| Field | Required | What |
|---|---|---|
| `path` | yes | Route path (web) or screen key (native). Used to identify the screen. |
| `name` | yes | Human-readable name surfaced in the manifest. |
| `sourceFile` | yes | Repo-relative path to the file that implements the screen. |
| `group` | no | Logical grouping for the sidebar in long manifests. |
| `figma.light` / `figma.dark` | no | Figma URLs for the screen variants. |
| `figma.other` | no | `Record<string, string>` for any additional frames (e.g. mobile, RTL). |
| `figmaDumps.light` / `figmaDumps.dark` | no | Filenames inside `figmaDumpsDir`. The agent reads these offline. |
| `description` | no | One-line summary for the agent. Especially useful for screens whose name alone is ambiguous. |

## Tips

- **Start narrow on `writePaths`.** Add the first folder you want polished, send a few tasks, then widen the list. A small allowlist means fewer surprise diffs to review.
- **Skip `readPaths` at first.** Letting the agent read broadly inside your repo is usually what you want -- it has to ground itself in your code to write the right thing.
- **Use `systemPromptAppend` for conventions, not for laundry lists.** A short note about your brand color and font wins more than a long checklist the agent will sometimes skim.
