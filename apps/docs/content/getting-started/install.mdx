## Pick the bits you need

omniUI is published as eight small packages. Pull only what you need; nothing else gets installed.

```bash
# The minimum useful set
npm install @omniui/core @omniui/styles @omniui/primitives
```

| Package | Install when | Dev dep? |
|---|---|---|
| `@omniui/core` | Always (tokens, types, hooks). The other packages depend on it. | runtime |
| `@omniui/styles` | Always (ThemeProvider + useTheme). Wraps your app once. | runtime |
| `@omniui/primitives` | You need Box, Text, Stack, Flex, Input, Button. | runtime |
| `@omniui/icons` | You need an Icon component + registry. | runtime |
| `@omniui/layouts` | You want AppShell, Sidebar, ScreenContainer. | runtime |
| `@omniui/navigator` | You want defineRoutes + a web/native adapter. | runtime |
| `@omniui/comps` | You want Modal, Dropdown (Tooltip + Tabs coming). | runtime |
| `@omniui/mcp` | You want coding agents to install components by name. | tooling |

A single line for the lot:

```bash
npm install @omniui/core @omniui/styles @omniui/primitives @omniui/icons \
  @omniui/layouts @omniui/navigator @omniui/comps
```

`@omniui/mcp` is a CLI you install once and run as an MCP server, not a runtime import:

```bash
npm install --save-dev @omniui/mcp
```

## Wrap your app in ThemeProvider

```tsx
import { ThemeProvider } from "@omniui/styles";

export default function App() {
  return (
    <ThemeProvider>
      <YourApp />
    </ThemeProvider>
  );
}
```

That's it. The provider sets CSS custom properties on `documentElement`, persists the active mode to `localStorage`, and follows the OS preference until the user explicitly toggles.

## Use a primitive

```tsx
import { Box, Text, Button } from "@omniui/primitives";
import { useTheme } from "@omniui/styles";

export function Hello() {
  const { toggleTheme } = useTheme();
  return (
    <Box surface="primary" padding="lg" radius="lg">
      <Text size="lg" weight="bold">Hello.</Text>
      <Button onClick={toggleTheme}>Flip theme</Button>
    </Box>
  );
}
```
